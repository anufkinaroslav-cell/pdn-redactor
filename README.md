# Редактор персональных данных в PDF

Локальный инструмент для удаления ФИО, ИНН, СНИЛС, паспортных данных, телефонов
и адресов из текстовых PDF-документов. Создан для соответствия 152-ФЗ при работе
с документами клиентов.

## Гарантия офлайна

Вся обработка файлов происходит **в браузере, на вашем компьютере**. Файл читается
через File API прямо в память вкладки, обрабатывается локальным JavaScript-кодом
(библиотеки pdf.js, pdf-lib и Tesseract.js для распознавания сканов лежат в
`vendor/`, ничего не подгружается из интернета) и сохраняется на диск через обычное
скачивание браузера. Ни один байт содержимого документов никуда не отправляется —
ни на внешние серверы, ни даже на сервер, с которого открыта страница (будь то
локальный `start.bat` или сайт на GitHub Pages — в обоих случаях сервер только
отдаёт статические файлы).

Единственный момент, когда требовался интернет — разовая загрузка библиотек pdf.js,
pdf-lib и Tesseract.js (вместе с офлайн-моделью русского языка) при настройке
проекта. Дальше можно работать без сети вовсе.

## Как запустить

### Вариант 1 — онлайн, без установки

Откройте сайт по ссылке: https://anufkinaroslav-cell.github.io/pdn-redactor/

Работает точно так же, как локальная версия — сайт статический, обработка
файлов происходит целиком в браузере, ничего никуда не отправляется.

### Вариант 2 — локально, на своём компьютере

1. Дважды кликните `start.bat`.
2. Откроется браузер на `http://localhost:8743`.

Требуется установленный Python (используется только как нулевая зависимость для
локального веб-сервера — сам он не участвует в обработке данных).

### Использование

Выберите один или несколько PDF-файлов, отметьте нужные типы данных, нажмите
«Обработать». Для каждого файла появится ссылка «Скачать: ИмяФайла_redacted.pdf» —
сохраните результат на диск.

## Как устроена редакция

Каждая страница PDF рендерится в изображение, найденные персональные данные
закрашиваются сплошными чёрными блоками прямо на пикселях этого изображения, после
чего итоговый PDF собирается заново только из таких изображений. Исходный
текстовый слой в результат не попадает вообще — скопировать текст из-под чёрных
блоков (или из любого другого места страницы) невозможно, так как текста там
больше нет физически.

Побочный эффект: весь текст на странице (не только скрытые данные) перестаёт быть
выделяемым и копируемым — это неизбежная плата за гарантированную необратимость
удаления.

## Отсканированные документы (OCR)

Если у страницы нет текстового слоя вообще (документ отсканирован или сфотографирован),
приложение автоматически распознаёт текст на картинке через встроенный офлайн-OCR
(Tesseract.js с моделью русского языка, тоже работает только в браузере). Это заметно
медленнее обычной обработки и зависит от качества скана — чем хуже скан (наклон, шум,
низкое разрешение, рукописный текст), тем больше риск пропустить часть данных или
ошибиться в паре символов. Для сканов рекомендуется обязательно проверять результат
глазами перед отправкой.

## Ограничения

- Обычные текстовые PDF (где текст можно выделить мышью) обрабатываются напрямую и
  надёжнее. Отсканированные документы — через OCR (см. выше), с меньшей гарантией.
- Распознавание ИНН, СНИЛС, паспорта и телефона — по устойчивым числовым форматам,
  довольно надёжное.
- Распознавание **ФИО и адресов** — эвристическое (по заглавным буквам, окончаниям
  отчеств, словарю имён и ключевым словам), без ИИ/NER-моделей. Возможны как
  пропуски (нестандартные форматы имён), так и лишние срабатывания. Поскольку
  обработка полностью автоматическая (без ручной проверки перед сохранением),
  для важных документов рекомендуется один раз открыть результат и убедиться, что
  всё нужное закрашено.
